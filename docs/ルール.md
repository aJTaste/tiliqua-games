あります。今のプロジェクトから帰納できるルールと、コピー&ペーストで使える雛形をまとめます。

---

## ディレクトリ構造の定石

新しいページを追加するたびに、**必ずこの3箇所にファイルを作る**のがルールです。

```
app/
  [ページ名]/
    layout.tsx        ← メタデータ（タイトル・説明文）だけ書く
    page.tsx          ← UIの本体

components/
  [ページ名]/         ← そのページ専用のコンポーネント
    (HogeComponent.tsx など)

lib/
  [ページ名]/         ← ロジック・型定義（ゲームなら特に重要）
    types.ts
    logic.ts
```

**ゲームやツールの規模が小さい場合**は `components/` と `lib/` のフォルダを作らず、`page.tsx` 一本で書き始めて、「長くなってきたら分ける」で問題ありません。

## 雛形

### `app/[ページ名]/layout.tsx`

```tsx
// app/[ページ名]/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "ページタイトル",
  description: "このページの説明",
};

export default function Layout({
  children,
}: {
  children: React.ReactNode;
}) {
  return <>{children}</>;
}
```

> ここには `<html>` も `<body>` も書かない。`<>{children}</>` を返すだけ。メタデータの置き場所というのが唯一の役割。

---

### `app/[ページ名]/page.tsx`（フルスクリーン型）

ゲームや、スクロールしないツール向け。

```tsx
// app/[ページ名]/page.tsx
"use client";

import CategoryTitle from "@/components/ui/CategoryTitle";
import PageFooter from "@/components/ui/PageFooter";
import Button from "@/components/ui/Button";

export default function Page() {
  return (
    <main className="flex flex-col items-center h-[100dvh] bg-[#F8F9FA]">

      {/* ヘッダー */}
      <header className="flex-shrink-0 w-full max-w-xs flex justify-between items-end px-1 pt-4 pb-3">
        <CategoryTitle category="Games" title="ページ名" />
      </header>

      <div className="w-full max-w-xs border-t border-[#e8e8e8]" />

      {/* コンテンツ */}
      <div className="flex-1 flex flex-col items-center justify-center gap-4 w-full max-w-xs px-5 overflow-y-auto py-3">
        {/* ここにメインコンテンツを書く */}
      </div>

      <PageFooter />
    </main>
  );
}
```

### `app/[ページ名]/page.tsx`（スクロール型）

記事、設定画面、縦に長くなるツール向け。

```tsx
// app/[ページ名]/page.tsx
"use client";

import CategoryTitle from "@/components/ui/CategoryTitle";
import PageFooter from "@/components/ui/PageFooter";

export default function Page() {
  return (
    <main className="flex flex-col items-center min-h-dvh bg-[#F8F9FA]">

      {/* ヘッダー */}
      <header className="w-full max-w-xs flex flex-col px-5 pt-8 pb-5">
        <CategoryTitle category="Tools" title="ページ名" />
      </header>

      <div className="w-full max-w-xs border-t border-[#e8e8e8]" />

      {/* コンテンツ */}
      <div className="flex-1 w-full max-w-xs px-5 py-8 flex flex-col gap-8">
        {/* ここにメインコンテンツを書く */}
      </div>

      <PageFooter />
    </main>
  );
}
```

## レイアウトとスタイルのルール一覧

### フルスクリーンかスクロールか

| 用途 | クラス |
|---|---|
| ゲーム・スクロールしないツール | `h-[100dvh]` |
| 記事・縦に長いツール | `min-h-dvh` |

`100dvh` の `d` は「dynamic（動的）」の意味で、スマホのアドレスバーが表示・非表示で高さが変わる問題を解消してくれます。

---

### 色・テキストの定数

プロジェクト全体で使う色は固定されています。新しいページでも必ずこれを使ってください。

| 用途 | 値 |
|---|---|
| 背景 | `#F8F9FA` |
| メインテキスト | `#1a1a1a` |
| 薄いテキスト（ラベル等） | `#bbb` |
| さらに薄いテキスト（soon等） | `#ccc` / `#ddd` |
| 区切り線 | `#e8e8e8` |
| 成功・ストリーク色 | `#22c55e` |

---

### フォントの使い分け

| 用途 | クラス |
|---|---|
| 通常テキスト（デフォルト） | 何も書かなくてよい（`layout.tsx` で設定済み） |
| カラーコード・数値など等幅フォント | `font-mono-game` |

---

### ボタンの使い分け

| 用途 | 書き方 |
|---|---|
| 通常ボタン | `<Button className="text-xs tracking-[0.2em] px-3 py-1">` |
| トグルボタン（ON状態） | `<Button active={true} className="...">` |
| トグルボタン（OFF状態） | `<Button active={false} className="...">` |
| タブ切り替え（下線スタイル） | 素の `<button>` に Tailwind 条件クラスで書く |

---

### `app/page.tsx` への追加

新しいページが完成したら、ハブページのリストに追加します。

```tsx
// available: false → true に変えて href を設定するだけ
const games = [
  { label: "新しいゲーム名", href: "/新しいページ名", available: true },
  ...
];
```

## チェックリスト（新ページ追加時）

```
□ app/[ページ名]/layout.tsx を作った（メタデータのみ）
□ app/[ページ名]/page.tsx を作った
□ <html><body> を layout.tsx に書いていない
□ ルート要素が <main> になっている
□ CategoryTitle・PageFooter を使っている
□ 背景色が bg-[#F8F9FA] になっている
□ インラインスタイルを使っていない（動的な色値は除く）
□ app/page.tsx のリストに追加した
```

---

**動的な色値（例：ゲームで計算した色を `backgroundColor` に渡す場合）だけはインラインスタイルで問題ありません。** それ以外の固定値はすべて Tailwind クラスで書く、というのが判断基準です。
